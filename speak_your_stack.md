## WHEN THEY ASK: "What is the ReAct pattern?"

"ReAct stands for Reason plus Act. The agent reasons about what it needs to do,
selects a tool, executes it, observes the result, then reasons again. It loops
until it has enough information to give a final answer. I built a bare-metal
ReAct loop without LangChain — just the raw API and a while loop that checks
stop_reason. When stop_reason is tool_use, Claude wants to call a tool. When
it's end_turn, it's done. The key insight is that stop_reason drives the entire
control flow."

---

## WHEN THEY ASK: "What is MCP?"

"MCP stands for Model Context Protocol. It's a standard that Anthropic developed
for connecting AI models to external systems. You build an MCP server that exposes
tools with defined schemas. The client — Claude Desktop or your own API loop —
connects to that server and Claude can call those tools in its reasoning loop.

I built an MCP server in Python using FastMCP that runs on my control plane server.
It exposes tools for SSH execution, pgvector search, memory storage, and file reading.
The server runs over stdio transport via SSH — so Claude Desktop on my MacBook SSHes
into my server and spawns the Python process. No open ports required."

## WHEN THEY ASK: "What is a vector database and why does it matter for AI?"

"A vector database stores embeddings — high-dimensional numerical representations
of text, images, or other data. When you want to find semantically similar content,
you embed your query and run a nearest-neighbor search against the stored vectors.
It's much more powerful than keyword search because it understands meaning, not
just exact words.

I'm running pgvector on PostgreSQL on my control plane. I have 592 memory rows
indexed using mxbai-embed-large embeddings generated by Ollama on my GPU node.
Clawdbot uses it to search Project Ascension documentation and past session notes
using natural language queries."

## WHEN THEY ASK: "What's the difference between an agent and a chatbot?"

"A chatbot responds to one message at a time. An agent has a goal and takes
multiple steps to achieve it — calling tools, observing results, replanning,
and iterating until the goal is complete or it hits a stopping condition.

The key technical difference is the loop. A chatbot is one API call. An agent
is a while loop that continues until stop_reason is end_turn with no pending
tool calls. The agent also maintains state — it knows what tools it called,
what they returned, and uses that context in its next reasoning step."

## WHEN THEY ASK: "How do you handle context window limits?"

"I implement a rolling window with a summarization fallback. I track the number
of conversation turns. When history exceeds a threshold — I use 8 turns — I make
a separate API call asking Claude to summarize the conversation into 3-5 sentences,
then replace the full history with that summary. This keeps the context window
manageable while preserving the semantic content of the conversation."
